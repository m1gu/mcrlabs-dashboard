Despues de que el user envia un prompt, tu primera proceso antes de iniciar a hacer codigo es:
1. Analizar el problema o situacion.
2. Revisar el codigo existente, incluido schema de base de datos, tablas, campos.
3. Armar un plan para proponer al user.
4. Mostrar el plan al user para saber si esta de acuerdo. y solicitar su aprobacion para proceder o hacer algun ajuste al plan.
